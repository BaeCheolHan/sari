## 2) 이번 턴에서 추가한 미커밋 변경 (중요)
아래 3개 파일은 **아직 커밋되지 않음**:
- `src/sari/mcp/server.py`
- `tests/unit/mcp/test_mcp_stdio_framed_transport.py`
- `uv.lock`

### 변경 목적
- `mcp__sari`에서 간헐적으로 보이던 `Transport closed`의 근본 원인 완화

### 근본 원인
- `run_stdio_streams()` 루프에서 `server.handle_request()` 예외가 상위로 전파되면 stdio 프로세스 자체가 종료됨
- 클라이언트에서는 이 종료를 `tools/call failed: Transport closed`로 관측

### 코드 수정 내용
- `src/sari/mcp/server.py`
  - `tools/call` 내부 예외를 pack1 에러로 변환(`ERR_MCP_TOOL_INTERNAL`)
  - stdio 루프에서 `handle_request()` 예외를 `-32603 internal error` 응답으로 변환하고 루프 유지
- `tests/unit/mcp/test_mcp_stdio_framed_transport.py`
  - `handle_request`가 예외를 던져도 프로세스 종료 대신 internal error 응답 반환하는 회귀 테스트 추가

### 해당 변경 검증
- `uv run pytest -q tests/unit/mcp/test_mcp_stdio_framed_transport.py -k handler_raises`
- `uv run pytest -q tests/unit/mcp/test_mcp_stdio_framed_transport.py tests/unit/mcp/test_mcp_server_protocol.py tests/unit/misc/test_no_silent_exception_policy.py`
- 결과: 모두 통과
