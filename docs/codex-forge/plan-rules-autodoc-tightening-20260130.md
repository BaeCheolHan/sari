# 규칙 강화(자동 문서화) 계획 (2026-01-30)

## 배경
- 문서/교훈/상태 기록이 누락되기 쉬움
- 세션 종료 시 일관된 산출물 관리가 필요

## 목표
- 특정 트리거에서 자동으로 계획/교훈/상태 문서를 남기도록 규칙을 명확화
- 세션 종료 시 산출물 경로와 상태 업데이트 여부를 항상 남기도록 유도

## 비목표
- 기존 개발 흐름을 과도하게 느리게 만드는 강제 절차
- 모든 요청에 문서화를 강제

## 규칙 강화 제안
1) **트리거 기반 자동 기록**
   - “계획/분석/리뷰/개선 제안/설계” 요청 시:
     - 계획 문서 1개 + lessons 1건 + 상태 기록 여부 확인
2) **세션 종료 체크리스트**
   - 요약 + 산출물 경로 + 상태 업데이트 여부
3) **예외 규정**
   - 단순 질의응답/소규모 설명은 문서 생략 가능(명시)

## 적용 위치(가이드)
- `.codex/rules/00-core.md` (핵심 규칙)
- 필요 시 `.codex/AGENTS.md` 또는 quick-start에 요약 문구 추가

## 스케일/예상 변경

## 리스크/대응
